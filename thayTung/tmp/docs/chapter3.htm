Module 3:       Setting up a Project

3.1.        Create a new Project

To start working on I2G, users need to create a new project as the first step. All of the input data, processed data and interpreted results will be saved in the project.

Step 1.         In the Home tab, click New Project (1). Input the project information.

Step 2.         Click OK (2). A new project will appear in the project (Figure 1.1).

Figure 3‑1 New Project dialog

3.5.        Share Project

You can share your own i2G’s project with any groups that are available in your organization within the i2G platform.

When you share the project, you can control whether people can edit, delete or view only.

Step 1.         In the Home tab, click Share Project. The Sharing Configuration dialog pops up.

Step 2.         The left-hand side table shows you all available Groups in your organization. Choose one or more groups to share a project by clicking on the checkbox next to each group name, this one will turn red.

Figure 3‑6 Sharing Configuration dialog

Step 3.         To choose what people can do with your project, focus on a shared Group name. The bottom table Permission lists all of the main components of your project. You can apply permission for a specific component including Well, Dataset, Curve, Zone Set, Zone, Log-plot, Log Track, Shading, Line, Marker, Annotation, Reference Curve, Depth Track, Image Track, Object Track, Object of Object Track, Zone track, Histogram, Cross-plot, Point Set, polygon, Regression Line, Ternary, User Define Line, Combined Box.

Tip: when you focus on a group, all the members of this will be listed on the right-side table as well

I2G provides 4 levels to share project:

·      Get: people can view but can’t change. The default is checked.

·      Create: people can create new data but can’t change your origin data. The default is checked.

·      Update: people can make changes to your origin data. The default is checked.

·      Delete: people can delete data. The default is unchecked.

2 global actions on the project database can be shared: Import and Export data.

Step 4.         Click Ok to start sharing a project.

Notes:

-       When anyone makes change data in a shared project, members will immediately be able to see and use the data inside.

-       If you permanently delete a shared project that you own, other collaborators can’t access as well.

-        When an administrator adds to or removes members from the group, they will gain or lose permission to the projects that the group can access.
